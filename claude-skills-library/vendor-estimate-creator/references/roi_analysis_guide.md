# ROI分析とビジネスケース作成ガイド

## Overview

このガイドは、システム開発プロジェクトのROI（投資対効果）を分析し、顧客向けに説得力のあるビジネスケースを作成するための方法を提供します。

---

## ROI分析の目的

1. **投資判断**: プロジェクトに投資する価値があるかを定量的に評価
2. **優先順位付け**: 複数のプロジェクト候補から優先順位を決定
3. **ステークホルダー説得**: 経営層、財務部門への説得材料
4. **プロジェクト正当化**: プロジェクト承認のための根拠
5. **効果測定**: プロジェクト完了後の効果検証

---

## 主要な財務指標

### 1. ROI（Return on Investment: 投資利益率）

**定義**: 投資に対してどれだけの利益が得られるかを示す指標

**計算式**:
```
ROI (%) = ((総利益 - 総投資額) / 総投資額) × 100
```

**例**:
```
システム開発投資: 1,000万円
年間コスト削減: 300万円
期間: 5年間
総利益 = 300万円 × 5年 = 1,500万円

ROI = ((1,500万円 - 1,000万円) / 1,000万円) × 100 = 50%
```

**解釈**:
- ROI > 0%: 投資により利益が得られる（承認の候補）
- ROI < 0%: 投資により損失が発生（承認困難）
- 一般的な目標: ROI 20-30%以上（3-5年間）

**メリット**:
- シンプルで理解しやすい
- 他の投資案と比較しやすい

**デメリット**:
- 時間価値（貨幣の現在価値）を考慮しない
- 投資期間が異なるプロジェクトの比較が難しい

---

### 2. NPV（Net Present Value: 正味現在価値）

**定義**: 将来のキャッシュフローを現在価値に割り引いた合計

**計算式**:
```
NPV = Σ (各年のキャッシュフロー / (1 + 割引率)^年) - 初期投資額
```

**例**:
```
初期投資: 1,000万円
年間便益: 300万円（5年間）
割引率: 10%

NPV = -1,000万円
      + 300万円/(1.10)^1
      + 300万円/(1.10)^2
      + 300万円/(1.10)^3
      + 300万円/(1.10)^4
      + 300万円/(1.10)^5

    = -1,000万円 + 272.7万円 + 247.9万円 + 225.4万円 + 204.9万円 + 186.3万円
    = 137.2万円
```

**解釈**:
- NPV > 0: プロジェクトは価値を生み出す（承認推奨）
- NPV = 0: プロジェクトは投資額と同等の価値（中立）
- NPV < 0: プロジェクトは価値を失う（承認非推奨）

**割引率の設定**:
- 企業のWACC（加重平均資本コスト）: 通常8-12%
- 低リスクプロジェクト: 8-10%
- 標準的プロジェクト: 10-12%
- 高リスクプロジェクト: 15-20%

**メリット**:
- 貨幣の時間価値を考慮
- 異なる期間のプロジェクト比較が可能
- 財務理論に基づいた正確な指標

**デメリット**:
- 割引率の設定が難しい
- 計算が複雑

---

### 3. IRR（Internal Rate of Return: 内部収益率）

**定義**: NPVをゼロにする割引率（プロジェクトの実質的な利回り）

**計算式**:
```
0 = Σ (各年のキャッシュフロー / (1 + IRR)^年) - 初期投資額
```

**例**:
```
初期投資: 1,000万円
年間便益: 300万円（5年間）

IRR = 15.24%

（NPVがゼロになる割引率を試行錯誤で求める）
```

**解釈**:
- IRR > WACC（企業の資本コスト）: 承認推奨
- IRR < WACC: 承認非推奨
- 一般的な目標: IRR 15-20%以上

**メリット**:
- 割引率を計算してくれる
- パーセンテージで理解しやすい
- 企業の資本コストと直接比較可能

**デメリット**:
- 複数のIRRが存在する場合がある
- プロジェクトの規模を考慮しない

---

### 4. Payback Period（投資回収期間）

**定義**: 初期投資を回収するまでにかかる期間

**計算式**:
```
累積キャッシュフローが初期投資を超える時点

単純回収期間 = 初期投資額 / 年間便益
```

**例**:
```
初期投資: 1,000万円
年間便益: 300万円

回収期間 = 1,000万円 / 300万円 = 3.33年
```

**解釈**:
- 短いほど良い（リスクが低い）
- 一般的な目標: 3年以内

**メリット**:
- シンプルで理解しやすい
- リスク評価に有用（早期回収=低リスク）

**デメリット**:
- 回収後のキャッシュフローを考慮しない
- 貨幣の時間価値を考慮しない（単純回収期間の場合）

## システム開発におけるベネフィット（便益）の分類

### 1. 定量的ベネフィット（Hard Benefits）

#### コスト削減

| ベネフィット項目 | 計算方法 | 例 |
|----------------|---------|-----|
| **人件費削減** | 削減工数 × 時間単価 | 月100時間削減 × 5,000円/時間 × 12ヶ月 = 600万円/年 |
| **外注費削減** | 削減した外注金額 | 年間データ入力外注500万円 → 自動化により0円 = 500万円/年 |
| **IT運用コスト削減** | 旧システムコスト - 新システムコスト | サーバー維持費1,000万円/年 → クラウド化で600万円/年 = 400万円/年削減 |
| **ライセンス費削減** | 削減したライセンス費用 | 旧ツール200万円/年 → 新ツール50万円/年 = 150万円/年削減 |
| **エラー修正コスト削減** | エラー件数削減 × 修正コスト | 月50件 → 月5件、修正コスト5万円/件 → 45件×5万円×12ヶ月 = 2,700万円/年 |
| **在庫コスト削減** | 在庫削減額 × 保管コスト率 | 在庫5億円削減 × 保管コスト率5% = 2,500万円/年 |

#### 売上増加

| ベネフィット項目 | 計算方法 | 例 |
|----------------|---------|-----|
| **新規顧客獲得** | 新規顧客数 × 平均LTV | 月10顧客増 × LTV 50万円 × 12ヶ月 = 6,000万円/年 |
| **顧客単価向上** | 単価増加額 × 顧客数 | 1,000円増 × 1万顧客 × 12ヶ月 = 1億2,000万円/年 |
| **クロスセル・アップセル** | 追加売上 | ECサイトレコメンド機能により売上3%増 → 売上10億円×3% = 3,000万円/年 |
| **機会損失の削減** | 削減した機会損失額 | ダウンタイム年10時間 → 年1時間、機会損失100万円/時間 → 900万円/年 |

#### 生産性向上

| ベネフィット項目 | 計算方法 | 例 |
|----------------|---------|-----|
| **処理時間短縮** | 短縮時間 × 時間単価 × 処理回数 | 1処理10分短縮 × 3,000円/時間 × 月1,000回 × 12ヶ月 = 600万円/年 |
| **スループット向上** | 追加処理件数 × 単価 | 処理能力20%向上 → 追加売上 = 1,000万円/年 |

---

### 2. 定性的ベネフィット（Soft Benefits）

| ベネフィット項目 | 評価方法 | 定量化のアプローチ |
|----------------|---------|-----------------|
| **顧客満足度向上** | NPS、CSATスコア | CSAT 7.0 → 8.5: 解約率5%削減 → 保持収益500万円/年 |
| **従業員満足度向上** | eNPS、離職率 | 離職率15% → 10%: 採用・育成コスト削減 → 1,000万円/年 |
| **ブランド価値向上** | ブランド調査スコア | - |
| **コンプライアンス強化** | リスク低減 | 個人情報漏洩リスク低減 → 潜在的損失1億円 × 発生確率5%削減 = 500万円/年相当 |
| **意思決定スピード向上** | 意思決定時間短縮 | 経営会議の意思決定時間50%短縮 → 機会損失削減（定量化は困難） |
| **イノベーション促進** | 新サービス創出数 | - |

**定性的ベネフィットの扱い**:
- 可能な限り定量化を試みる
- 定量化困難な場合は、ナラティブ（物語）で価値を説明
- 「戦略的重要性」として位置づける

## ビジネスケース作成の手順

### Step 1: 現状分析（As-Is）

**収集すべき情報**:
1. 現在のコスト
   - 人件費（工数 × 単価）
   - システム運用コスト
   - 外注費
   - エラー対応コスト

2. 現在の課題
   - 処理時間
   - エラー率
   - 顧客満足度
   - 従業員満足度

3. 機会損失
   - ダウンタイムによる損失
   - 非効率による損失
   - 新規ビジネス機会の逸失

**例**:
```markdown
## 現状（As-Is）

### 業務プロセス
- 受注処理: 平均30分/件、月1,000件
- 手作業によるデータ入力: エラー率5%
- エラー修正: 平均1時間/件、月50件

### 現在のコスト
- 受注処理人件費: 30分 × 1,000件 × 3,000円/時間 × 12ヶ月 = 1,800万円/年
- エラー修正人件費: 1時間 × 50件 × 3,000円/時間 × 12ヶ月 = 180万円/年
- システム運用コスト: 500万円/年
- **合計: 2,480万円/年**

### 主要課題
1. 処理時間が長く、リードタイム24時間
2. エラー率5%により顧客満足度低下（CSAT 6.5/10）
3. 手作業が多く、従業員のモチベーション低下
```

---

### Step 2: 将来像定義（To-Be）

**定義すべき項目**:
1. 新システムの機能
2. 期待される改善効果
3. 削減されるコスト
4. 増加する売上

**例**:
```markdown
## 将来像（To-Be）

### 新システムの機能
- 自動受注処理システム
- AIによるデータ入力・検証
- リアルタイム在庫連携
- 顧客ポータル

### 期待される改善効果
- 受注処理時間: 30分 → 5分（83%削減）
- エラー率: 5% → 0.5%（90%削減）
- リードタイム: 24時間 → 1時間
- 顧客満足度: CSAT 6.5 → 8.5

### コスト削減
- 受注処理人件費: 5分 × 1,000件 × 3,000円/時間 × 12ヶ月 = 300万円/年（▲1,500万円）
- エラー修正人件費: 1時間 × 5件 × 3,000円/時間 × 12ヶ月 = 18万円/年（▲162万円）
- システム運用コスト: 300万円/年（▲200万円）
- **合計削減: 1,862万円/年**

### 売上増加
- 処理スピード向上により新規受注対応能力20%向上 → 売上2%増加
- 売上10億円 × 2% = 2,000万円/年
```

---

### Step 3: 投資額の算出

**初期投資**:
- システム開発費
- ハードウェア・インフラ費用
- ライセンス費用（初期）
- 導入コンサルティング費用
- トレーニング費用
- データ移行費用

**運用コスト（年間）**:
- システム保守費用（開発費の15-20%）
- クラウド利用料
- ライセンス費用（年間）
- 運用人件費

**例**:
```markdown
## 投資額

### 初期投資
- システム開発費: 3,000万円
- インフラ構築費: 300万円
- ライセンス費（初期）: 200万円
- 導入コンサル: 150万円
- トレーニング: 100万円
- データ移行: 250万円
- **合計初期投資: 4,000万円**

### 年間運用コスト
- 保守費用: 450万円/年（開発費の15%）
- クラウド利用料: 120万円/年
- ライセンス費: 50万円/年
- **合計年間コスト: 620万円/年**
```

### Step 4: キャッシュフロー分析

**5年間のキャッシュフロー計算**:

| 年 | 初期投資 | 年間便益 | 年間運用コスト | 純便益 | 累積キャッシュフロー |
|----|---------|---------|--------------|--------|------------------|
| 0年 | ▲4,000万円 | 0万円 | 0万円 | ▲4,000万円 | ▲4,000万円 |
| 1年 | 0万円 | 3,862万円 | ▲620万円 | 3,242万円 | ▲758万円 |
| 2年 | 0万円 | 3,862万円 | ▲620万円 | 3,242万円 | 2,484万円 |
| 3年 | 0万円 | 3,862万円 | ▲620万円 | 3,242万円 | 5,726万円 |
| 4年 | 0万円 | 3,862万円 | ▲620万円 | 3,242万円 | 8,968万円 |
| 5年 | 0万円 | 3,862万円 | ▲620万円 | 3,242万円 | 12,210万円 |

**年間便益の内訳**:
- コスト削減: 1,862万円/年
- 売上増加: 2,000万円/年
- **合計便益: 3,862万円/年**

### Step 5: 財務指標の計算

#### ROI計算

```
総投資額 = 4,000万円
5年間の総便益 = 3,862万円 × 5年 = 19,310万円
5年間の総運用コスト = 620万円 × 5年 = 3,100万円
純利益 = 19,310万円 - 3,100万円 - 4,000万円 = 12,210万円

ROI = (12,210万円 / 4,000万円) × 100 = 305%
```

#### NPV計算（割引率10%）

```
NPV = -4,000万円
      + 3,242万円 / (1.10)^1
      + 3,242万円 / (1.10)^2
      + 3,242万円 / (1.10)^3
      + 3,242万円 / (1.10)^4
      + 3,242万円 / (1.10)^5

    = -4,000万円 + 2,947万円 + 2,679万円 + 2,436万円 + 2,214万円 + 2,013万円
    = 8,289万円
```

#### IRR計算

```
IRR = 約72%

（NPVがゼロになる割引率を試行錯誤で求める）
```

#### 回収期間

```
累積キャッシュフロー:
- 1年後: ▲758万円
- 2年後: 2,484万円（回収完了）

回収期間 = 1年 + (758万円 / 3,242万円) = 1.23年
```

### Step 6: 感度分析（Sensitivity Analysis）

**リスクシナリオ**:

| シナリオ | 前提 | NPV | ROI | 回収期間 |
|---------|------|-----|-----|---------|
| **最良ケース** | 便益+20%、コスト-10% | 12,500万円 | 420% | 0.9年 |
| **標準ケース** | 計画通り | 8,289万円 | 305% | 1.23年 |
| **悲観ケース** | 便益-20%、コスト+10% | 3,800万円 | 185% | 1.7年 |
| **最悪ケース** | 便益-30%、コスト+20% | 1,500万円 | 95% | 2.2年 |

**感度分析の解釈**:
- 悲観ケースでもROI 185%、NPV 3,800万円とプラス
- 最悪ケースでもROI 95%で投資回収可能
- **結論**: リスクを考慮してもプロジェクトは実行価値あり

---

### Step 7: リスクと前提条件の明示

**主要前提条件**:
1. 受注件数が月1,000件で安定
2. 人件費単価3,000円/時間で計算
3. 新システムによりエラー率が0.5%に低下
4. 導入後6ヶ月で効果発現（1年目は6ヶ月分の効果）

**主要リスク**:
1. 導入遅延リスク（確率: 中、影響: 高）
   - 対策: アジャイル開発、段階的リリース
2. 効果発現遅延リスク（確率: 中、影響: 中）
   - 対策: 変更管理、トレーニング強化
3. 受注件数減少リスク（確率: 低、影響: 高）
   - 対策: 市場分析、代替収益源の検討

## ROI分析のベストプラクティス

### 1. 保守的に見積もる

- **ベネフィット**: 控えめに見積もる（達成確度80%以上）
- **コスト**: 余裕を持って見積もる（コンティンジェンシー15-20%）
- **効果発現**: 段階的に発現すると仮定（1年目50%、2年目100%）

### 2. 定量化できるベネフィットに絞る

- 不確実な定性的ベネフィットは「追加価値」として別記
- 財務指標の計算には確実な定量的ベネフィットのみ使用

### 3. 複数のシナリオを提示

- 最良、標準、悲観、最悪の4シナリオ
- 悲観ケースでも投資価値があることを示す

### 4. 時間価値を考慮

- NPVとIRRを必ず計算
- 単純なROIだけでは不十分

### 5. ステークホルダーに応じた説明

- **経営層**: エグゼクティブサマリー、財務指標、戦略的価値
- **財務部門**: NPV、IRR、キャッシュフロー、感度分析
- **IT部門**: 技術的実現可能性、運用コスト
- **事業部門**: 業務改善効果、顧客満足度向上

## まとめ

**ROI分析の成功要因**:
1. 現状の正確な把握（As-Is分析）
2. 現実的な将来像（To-Be定義）
3. 保守的なベネフィット見積もり
4. 包括的な投資額算出
5. 複数の財務指標による評価（ROI、NPV、IRR、回収期間）
6. 感度分析によるリスク評価
7. ステークホルダーに応じた説明

**ROI分析は単なる数字の羅列ではなく、プロジェクトの価値を説得力を持って伝えるストーリー**です。
