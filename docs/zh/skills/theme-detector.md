## 1. 概述

Theme Detector 识别当下哪些市场主题(AI 与半导体、清洁能源、黄金、网络安全、国防等)正在获得或失去动量。它不分析个股,而是在主题层面运作 —— 聚合每个主题内全部成分行业与 ETF 的表现、成交量与宽度数据。

**它解决什么:**
- 跟上哪些市场叙事正在驱动资金流动
- 把早期主题与接近衰竭的拥挤交易区分开
- 以与多头主题同等的敏感度识别空头主题(承压板块)
- 为主题投资机会提供结构化、量化的视角

**三维评分模型:**

| 维度 | 范围 | 衡量内容 |
|------|------|----------|
| **主题热度** | 0-100 | 主题强度:动量、成交量强度、上升趋势比率、宽度 |
| **生命周期成熟度** | 萌芽 / 加速 / 趋势 / 成熟 / 衰竭 | 主题发展到何种程度:持续时间、RSI 极端度、估值、ETF 增殖 |
| **置信度** | 低 / 中 / 高 | 检测可靠性:量化宽度结合叙事确认 |

**追踪 14+ 个主题:** AI 与半导体、清洁能源与 EV、网络安全、云计算与 SaaS、生物科技与基因组、基建与建筑、黄金与贵金属、国防与航空航天、能源(油气)、必需消费与防御、金融与银行、房地产、区块链与加密、医疗创新。

## 4. 工作原理

1. **行业数据收集** —— 扫描约 145 个 FINVIZ 行业的表现、成交量与个股级指标(经 FINVIZ Elite API 或公开抓取)。
2. **主题分类** —— 用 `cross_sector_themes.md` 中的定义把行业映射到跨板块主题。每个主题需要最少数量的成分行业显示活跃,才会被分类。
3. **热度评分** —— 由四个子分量计算主题热度(0-100):
   - **动量强度** —— 多时间框架表现加权
   - **成交量强度** —— 当前成交量 vs 历史均值
   - **上升趋势比率** —— 成分股中处于技术上升趋势的比例
   - **宽度** —— 方向一致加权收益的匹配行业占比(LEAD 主题为正、LAG 主题为负)。是行业级参与率,而非个股级
4. **生命周期评估** —— 由以下因素判定成熟阶段:
   - **持续时间分** —— 主题活跃多久
   - **极端度聚集** —— RSI 读数在极端处的集中度
   - **价格极值饱和** —— 处于 52 周高/低点的股票
   - **估值溢价** —— 相对历史常态的 P/E 溢价(需 FMP)
   - **ETF 增殖** —— 主题 ETF 数量(越多 = 越成熟/越拥挤)
5. **方向检测** —— 基于加权行业表现、上升趋势比率与成交量确认,把每个主题分类为多头、空头或中性。
6. **叙事确认** —— 对前 5 个主题,Claude 执行 WebSearch 查询,验证量化信号是否与现实新闻和分析师覆盖一致。

**为何有用:** 无需任何 API 订阅,就能最快地鸟瞰资金在市场各处的流向。

### 示例 3:生命周期评估

**提示词:**
```
AI 主题还处于早期还是变拥挤了?
```

**Claude 分析什么:**
- **持续时间:** AI 走强了几个月?
- **RSI 极端度:** AI 股票是否聚集在超买水平?
- **ETF 增殖:** AI 主题 ETF 数量(SMH、SOXX、AIQ、BOTZ、CHAT)—— ETF 越多 = 散户参与越多 = 越成熟
- **估值溢价:** 半导体/软件股当前 P/E vs 历史均值

**解读:**
- **萌芽(成熟度 0-20):** 参与者少、ETF 数量低、主题刚起步 —— 最佳入场时机
- **加速(20-40):** 参与度增长、部分 ETF 已发行、媒体报道增加
- **趋势(40-60):** 动量强劲、分析师覆盖广、主流主题
- **成熟(60-80):** 参与广泛、多只 ETF、共识交易、估值被拉伸
- **衰竭(80-100):** ETF 增殖达到顶峰、估值极端、逆向信号出现

### 示例 4:空头主题识别

**提示词:**
```
现在市场上最强的空头主题有哪些?
```

**发生了什么:** 检测器识别加权表现为负、上升趋势比率低于 50% 且成交量确认派发(抛压)的主题。它们用与多头主题相同的 热度/生命周期/置信度 评分呈现,但使用反向指标。

**为何有用:** 空头主题揭示哪些板块应回避、对冲,或在生命周期到达衰竭时关注均值回归机会。

---

### 示例 5:热度评分解读

热度评分综合四个子分量:

| 热度评分 | 解读 | 行动 |
|----------|------|------|
| 80-100 | 极强动量 + 成交量 + 宽度 | 若生命周期为萌芽/加速,可考虑敞口 |
| 60-79 | 扎实的主题动量、参与广泛 | 标准配置,监控生命周期 |
| 40-59 | 中等主题信号,可能在萌芽或消退 | 观察清单,等确认 |
| 20-39 | 弱主题信号 | 不配置,监控是否反转 |
| 0-19 | 无有意义的主题活动 | 忽略 |

**黄金组合:** 高热度 + 萌芽生命周期 = 最佳机会。主题动量强但尚未吸引人群。

> 高热度 + 成熟/衰竭生命周期 = 谨慎。动量强但交易拥挤、有反转风险。

## 6. 解读输出

检测器生成:
- `theme_detector_YYYY-MM-DD_HHMMSS.json` —— 供程序化使用的结构化数据
- `theme_detector_YYYY-MM-DD_HHMMSS.md` —— 人类可读报告

**报告章节:**

1. **主题仪表盘** —— 含热度、方向、生命周期与置信度列的头部主题表
2. **多头主题详情** —— 每个多头主题含:
   - 热度评分拆解(动量、成交量、上升趋势、宽度子分)
   - 含阶段分类的生命周期成熟度评估
   - 主题内表现最佳的行业
   - 代表性股票与代理 ETF
   - 来自 WebSearch 的叙事确认
3. **空头主题详情** —— 空头主题的相同结构
4. **全部主题摘要** —— 所有检测主题的完整排名表
5. **行业排名** —— 表现最佳与最差的个别行业
6. **板块上升趋势比率** —— 板块级宽度聚合(当有上升趋势数据时)
7. **方法说明** —— 评分模型的简要解释

**方向分类:**

方向由成分行业相对排名的多数表决决定:
1. 全部约 145 个行业按多时间框架动量评分排名
2. 上半部行业 = “多头”,下半部 = “空头”
3. 每个主题的方向是其匹配行业的多数表决

**显示映射:**
- “bullish” → **LEAD**(匹配行业的相对跑赢)
- “bearish” → **LAG**(匹配行业的相对跑输)
- LAG 主题可能有正的绝对收益 —— 它表示相对跑输,而非做空信号

## 7. 技巧与最佳实践

- **每周运行用于战略配置。** 主题趋势以周到月的时间框架变化。每日扫描只增加噪声而不改变结论。
- **把热度与生命周期结合做决策。** 单凭热度可能误导 —— 热度 90 且生命周期衰竭的主题可能即将反转。
- **用代理 ETF 快速建立敞口。** 每个主题都含代理 ETF(例如 AI 与半导体的 SMH/SOXX、生物科技的 XBI/IBB)。它们提供即时、分散的主题敞口。
- **把 ETF 增殖当作逆向信号。** 当大量新主题 ETF 发行时,通常意味着周期晚期的散户参与。追踪某主题的 ETF 越多,它越可能接近衰竭。
- **与叙事交叉引用。** 量化高 + 叙事弱 是背离信号(有动量无信念)。量化低 + 叙事强 可能提示一个价格尚未跟上的萌芽主题。
- **公开模式足以做每周分析。** 公开模式下每行业约 20 只股票已捕捉主要参与者。Elite 模式对小盘主题或需要实时数据时增添价值。

## 9. 故障排查

### 公开模式执行慢

**预期行为:** 公开 FINVIZ 模式因限速(约 145 个行业、每次请求间隔 2.0 秒)需 5-8 分钟。

**修复:** 若速度关键,配置 FINVIZ Elite($39.50/月),用 `--finviz-api-key` 实现 2-3 分钟执行。

### 缺少 Python 依赖

```
ModuleNotFoundError: No module named 'pandas'
```

安装所有必需包:
```bash
pip install requests beautifulsoup4 lxml pandas numpy yfinance
```

### 某个特定主题“未检测到”

**原因:** 主题检测需要最少数量的活跃匹配行业。全局 `cross_sector_min_matches` 设置(默认 2,可在 themes.yaml 配置)控制此阈值。若有意义数据的行业太少,主题不会被分类。

**修复:** 对行业代表性狭窄的主题,这是预期行为。查看 `references/cross_sector_themes.md` 了解设计意图的最低值。FINVIZ Elite 提供更好覆盖,或可解决此问题。

### 公开模式数据有限

公开 FINVIZ 抓取每行业返回约 20 只股票(仅第一页)。这可能错过主题内的小盘信号。

**修复:** 如需全面分析,用返回每行业完整股票范围的 FINVIZ Elite。但对多数主题级决策,公开模式的大盘代表性已足够。

### 叙事确认不可用

基于 WebSearch 的叙事确认取决于网络连接与搜索结果质量。若 WebSearch 返回有限结果:
- 置信度默认仅采用量化评估
- 报告会将此标为数据质量限制
- 量化评分仍完全有效
