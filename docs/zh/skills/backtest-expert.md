# Backtest Expert
{: .no_toc }

## 1. 概述

Backtest Expert 用一套系统化、对抗性的方法评估交易策略。它问的不是“这个策略能赚多少?”,而是“什么能击破这个策略?”在悲观条件下幸存的策略,在实盘中起作用的可能性要高得多。

**它解决什么:**
- 用量化的 5 维评分框架取代主观的“看起来不错”式回测评估
- 检测提示回测脆弱或具有误导性的常见危险信号
- 给出清晰的 部署/打磨/放弃 裁决并附具体理由
- 强制对参数敏感度、样本量与执行成本做诚实评估
- 防范最危险的回测陷阱:曲线拟合、前视偏差与幸存者偏差

**核心能力:**
- 5 维评分(0-100):样本量、期望值、风险管理、稳健性、执行现实性
- 10 多种自动危险信号检测(交易过少、负期望、过度优化、回撤过大等)
- 基于综合分与危险信号严重度的 部署/打磨/放弃 裁决
- 参数稳健性评估(平台 vs 尖峰分析)
- 在典型估计 1.5-2 倍处做滑点与摩擦压力测试

<span class="badge badge-free">无需 API</span>

## 2. 前提条件

- **API 密钥:** 无需 —— 所有指标由用户提供
- **Python 3.9+:** 运行评估脚本所需
- **无额外 Python 依赖** —— 仅使用标准库
- **无需外部数据** —— 脚本根据你提供的数字评分

> Backtest Expert 是纯评估工具。你提供回测统计,它给它们评分。无需市场数据、无 API 调用、无需联网。

## 3. 快速开始

对 Claude 说:

```
我回测了一个财报跳空均值回归策略,8 年 150 笔交易,
胜率 62%,平均盈利 1.8%,平均亏损 1.2%,最大回撤 15%,3 个参数,
且测试了滑点。它得分如何?
```

或直接运行评估脚本:

```bash
python3 skills/backtest-expert/scripts/evaluate_backtest.py \
  --total-trades 150 \
  --win-rate 62 \
  --avg-win-pct 1.8 \
  --avg-loss-pct 1.2 \
  --max-drawdown-pct 15 \
  --years-tested 8 \
  --num-parameters 3 \
  --slippage-tested \
  --output-dir reports/
```

脚本在 5 个维度上评分,标记任何隐患,并给出 部署/打磨/放弃 裁决。这就是上手所需的一切。

---

## 4. 工作原理

1. **收集指标** —— 脚本接收 8 个输入:总交易数、胜率、平均盈利 %、平均亏损 %、最大回撤 %、测试年数、可调参数数量,以及是否建模了滑点。
2. **5 维评分** —— 每个维度在 0-20 区间独立评分:
   - **样本量(20):** 基于总交易数与每年交易密度。200+ 笔得分最高;<30 笔接近零。
   - **期望值(20):** 由盈利因子(胜率 × 平均盈利 / 亏率 × 平均亏损)得出。盈利因子 1.0 表示盈亏平衡;1.5+ 为健康。
   - **风险管理(20):** 最大回撤严重度与盈利因子共同决定此分。回撤低于 15% 得分最佳;超过 40% 接近零。
   - **稳健性(20):** 参数数量与测试年数。更少参数(4 个或更少)与更长测试期(10+ 年)得分最高。
   - **执行现实性(20):** 是否建模了滑点与摩擦。测试了滑点得满分;跳过则封顶。
3. **检测危险信号** —— 脚本检查 10 多种警告模式:交易过少(<30)、负期望、过度优化(7+ 参数)、极端回撤(>40%)、未测试滑点、测试期过短(<3 年)等。
4. **给出裁决** —— 基于综合分与危险信号严重度:
   - **部署(Deploy):** 高分、无关键危险信号。策略可实盘。
   - **打磨(Refine):** 中等分或非关键危险信号。核心逻辑可靠但需调整。
   - **放弃(Abandon):** 低分或关键危险信号。策略存在根本缺陷。
5. **输出报告** —— JSON 与 Markdown 文件保存到输出目录。

## 5. 使用示例

### 示例 1:评估财报跳空均值回归策略

**提示词:**
```
我测试了一个财报跳空策略:8 年 150 笔交易,胜率 62%,平均盈利 1.8%,
平均亏损 1.2%,最大回撤 15%,3 个参数,测试了滑点。给它评分。
```

**命令:**
```bash
python3 skills/backtest-expert/scripts/evaluate_backtest.py \
  --total-trades 150 \
  --win-rate 62 \
  --avg-win-pct 1.8 \
  --avg-loss-pct 1.2 \
  --max-drawdown-pct 15 \
  --years-tested 8 \
  --num-parameters 3 \
  --slippage-tested \
  --output-dir reports/
```

**为何有用:** 这是个扎实的策略 —— 样本量好、期望为正、回撤可控、参数少、且测试了执行成本。评估精确量化它有多扎实,并识别任何剩余隐患。

### 示例 2:为技术形态筛选器评分

**提示词:**
```
我的 VCP 筛选器在 5 年里找到 85 笔交易,胜率 48% 但平均盈利 3.5%、
平均亏损 1.5%。最大回撤 22%,5 个参数,尚未测试滑点。
```

**命令:**
```bash
python3 skills/backtest-expert/scripts/evaluate_backtest.py \
  --total-trades 85 \
  --win-rate 48 \
  --avg-win-pct 3.5 \
  --avg-loss-pct 1.5 \
  --max-drawdown-pct 22 \
  --years-tested 5 \
  --num-parameters 5 \
  --output-dir reports/
```

**为何有用:** 这个策略胜率低但盈亏比高(2.33:1)。评估判断其期望是否足以弥补,把缺失的滑点测试标为隐患,并指出中等的参数数量。

---

### 示例 3:危险信号检测(脆弱回测)

**提示词:**
```
我发现一个惊人的策略:2 年 25 笔交易,胜率 88%,平均盈利 4.2%,
平均亏损 0.8%,最大回撤 8%,9 个参数,未测试滑点。
```

**命令:**
```bash
python3 skills/backtest-expert/scripts/evaluate_backtest.py \
  --total-trades 25 \
  --win-rate 88 \
  --avg-win-pct 4.2 \
  --avg-loss-pct 0.8 \
  --max-drawdown-pct 8 \
  --years-tested 2 \
  --num-parameters 9 \
  --output-dir reports/
```

**为何有用:** 这个回测在尖叫“曲线拟合”。尽管收益看起来亮眼,评估会标记多个关键危险信号:交易过少(25 < 30)、参数过多(9 触发过度优化)、测试期过短(2 年)、未测试滑点。裁决是放弃,最好也只是强烈打磨。看起来“好得不真实”的结果,几乎总是如此。

---

### 示例 4:两个策略的并排对比

**提示词:**
```
对比策略 A(200 笔,胜率 55%,平均盈利 2.0%,平均亏损 1.5%,回撤 18%,
7 年,4 参数,测试滑点)vs 策略 B(120 笔,胜率 65%,平均盈利 1.2%,
平均亏损 0.9%,回撤 12%,7 年,6 参数,测试滑点)。
```

**发生了什么:** Claude 运行评估脚本两次,并在对比表中并排呈现评分。逐维度对比,整体裁决解释哪个策略更稳健及原因。

**为何有用:** 在两个策略间抉择时,主观“感觉”不可靠。5 维评分框架使对比客观,精确突出各策略的强弱所在。

---

### 示例 5:加大滑点的压力测试

**提示词:**
```
我的策略在标准滑点(0.05%)下有效。若滑点翻倍到 0.10% 会怎样?
```

**发生了什么:** Claude 引导你用悲观滑点假设(典型 1.5-2 倍)重跑回测。前后对比揭示策略对执行成本有多敏感。

**为何有用:** 许多回测在零或极小滑点下看起来盈利,却在现实执行成本下崩溃。在典型滑点 1.5-2 倍处做压力测试,能把真正的优势与幻觉区分开。

---

### 示例 6:参数稳健性评估

**提示词:**
```
我的止损设在 2.0%。若从 1.0% 到 3.0% 变动,策略还有效吗?
```

**发生了什么:** Claude 要求你在多个止损水平(1.0%、1.5%、2.0%、2.5%、3.0%)运行回测,并评估表现是呈稳定“平台”(稳健)还是狭窄“尖峰”(脆弱)。

**为何有用:** 只有止损恰好 2.0% 才有效的策略是曲线拟合;从 1.5% 到 3.0% 都有效的策略才有真正优势。追求平台,而非峰值。

---

### 示例 7:前推验证(Walk-Forward)

**提示词:**
```
我在 2018-2022 数据上优化。该如何用 2023-2025 数据验证?
```

**发生了什么:** Claude 带你走完前推流程:(1) 把 2018-2022 的参数原样应用于 2023-2025,(2) 对比样本内 vs 样本外表现。若样本外不到样本内的 50%,策略很可能过拟合。

**为何有用:** 前推分析是检测过拟合的黄金标准。参数在期间剧烈漂移,或样本外表现崩溃,都表明是噪声而非优势。

## 6. 解读输出

执行后,脚本生成含以下内容的 JSON 与 Markdown 报告:

1. **输入指标** —— 你提供的原始回测统计。
2. **派生指标** —— 计算值,包括盈利因子、每笔期望值与每年交易数。
3. **5 维评分** —— 每维 0-20 分并附简要说明:

| 维度 | 满分 | 衡量内容 |
|------|------|----------|
| 样本量 | 20 | 交易数与密度(笔/年) |
| 期望值 | 20 | 盈利因子与每笔优势 |
| 风险管理 | 20 | 回撤严重度与风险调整后收益 |
| 稳健性 | 20 | 参数数量与测试期长度 |
| 执行现实性 | 20 | 是否建模了滑点与摩擦 |

4. **危险信号** —— 检测到的问题列表,各含严重度(关键、警告或提示)与描述。
5. **综合分** —— 5 个维度之和(0-100)。
6. **裁决** —— 部署、打磨或放弃,附理由。

## 7. 技巧与最佳实践

- **把 80% 的时间用于试图击破策略。** 生成假设占 20% 精力;压力测试占 80%。若策略能在你最强的击破尝试中幸存,优势很可能是真的。
- **把“好得不真实”的结果当危险信号。** 80% 以上的胜率、极小回撤与完美择时,几乎总是前视偏差、幸存者偏差或曲线拟合的产物。
- **要求至少 100 笔交易。** 绝对下限是 30,但偏好 100+,200+ 提供高置信。小样本无法区分技能与运气。
- **跨多个市场体制测试。** 只在牛市有效的策略不是优势 —— 那是对方向的杠杆下注。要求在多数测试年份期望为正。
- **参数保持在 4 个或更少。** 每个参数都增加一个可拟合噪声的自由度。脚本把 7+ 参数标为过度优化。
- **始终测试滑点。** 不带 `--slippage-tested` 运行会封顶执行现实性维度。用典型滑点的 1.5-2 倍估计。
- **把想法生成与验证分开。** 直觉生成假设;验证必须纯数据驱动。

## 9. 故障排查

### 裁决为“放弃”但策略看起来盈利

**原因:** 策略可能收益为正但有关键结构性弱点:交易太少缺乏统计置信、参数太多(曲线拟合风险),或未测试执行成本。

**修复:** 针对具体危险信号。若样本量低,在更长期或更广范围测试;若参数太多,简化策略;若未测滑点,用典型摩擦的 1.5 倍重跑。

### 胜率高但评分平平

**原因:** 单凭高胜率不保证好分数。若平均盈利相对平均亏损很小(盈亏比差),尽管频繁获胜,期望也可能很弱。90% 胜率配 0.3% 平均盈利与 3.0% 平均亏损会产生负期望。

**修复:** 检查输出中的盈利因子。盈利因子低于 1.0 意味着系统在亏钱。期望值维度基于胜率 *与* 盈亏比的组合评分,而非仅凭胜率。

### 评分似乎太苛刻

**原因:** 评分框架刻意保守。它旨在防止部署脆弱策略,而非验证一厢情愿。

**修复:** 这是设计预期。理念是“找到最难被击破的策略”。得分 60-70 且裁决为打磨其实是好结果 —— 它意味着核心逻辑可靠,但需要危险信号中指出的具体改进。

### 只有 5 个参数却报“过度优化”

**原因:** 脚本把 7+ 参数标为过度优化。在 5-6 个参数时不会触发标志,但相比 4 个或更少参数的策略,稳健性维度分会降低。

**修复:** 若你认为这 5 个参数都必不可少,降低的分数是合理的 —— 即使每个参数都有明确用途,更多参数确实增加曲线拟合风险。考虑是否有参数可消除或固定为整数。

## 10. 参考资料

### CLI 参数

| 参数 | 必需 | 默认 | 说明 |
|------|------|------|------|
| `--total-trades` | 是 | -- | 回测中的交易数 |
| `--win-rate` | 是 | -- | 胜率百分比(例如 62 表示 62%) |
| `--avg-win-pct` | 是 | -- | 平均盈利交易百分比 |
| `--avg-loss-pct` | 是 | -- | 平均亏损交易百分比(正数) |
| `--max-drawdown-pct` | 是 | -- | 最大回撤百分比 |
| `--years-tested` | 是 | -- | 回测期年数 |
| `--num-parameters` | 是 | -- | 策略中可调参数数量 |
| `--slippage-tested` | 否 | `false` | 标志是否建模了滑点/摩擦 |
| `--output-dir` | 否 | `reports/` | JSON 与 Markdown 报告的输出目录 |

### 5 维评分概览

| 维度 | 分数 | 关键阈值 |
|------|------|----------|
| 样本量 | 0-20 | 200+ 笔 = 满分;<30 = 接近零 |
| 期望值 | 0-20 | 盈利因子 1.5+ = 健康;<1.0 = 负优势 |
| 风险管理 | 0-20 | 回撤 <15% = 最佳;>40% = 接近零 |
| 稳健性 | 0-20 | 4 个或更少参数 + 10+ 年 = 最佳;7+ 参数 = 过度优化标志 |
| 执行现实性 | 0-20 | 测试了滑点 = 满分;未测 = 封顶 |

### 裁决标准

| 裁决 | 含义 | 典型行动 |
|------|------|----------|
| 部署 | 在可接受表现下通过所有压力测试 | 进入模拟交易,再用小仓位实盘 |
| 打磨 | 核心逻辑可靠但需调参或更多测试 | 针对具体危险信号,再评估 |
| 放弃 | 未通过压力测试或依赖脆弱假设 | 停止开发,转向下一个假设 |

### 常见危险信号

| 危险信号 | 触发条件 |
|----------|----------|
| 交易过少 | 总交易 < 30 笔 |
| 负期望 | 盈利因子 < 1.0 |
| 过度优化 | 7+ 可调参数 |
| 极端回撤 | 最大回撤 > 40% |
| 测试期过短 | 测试 < 3 年 |
| 未测试滑点 | 未设 `--slippage-tested` |
