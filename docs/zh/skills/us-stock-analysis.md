## 1. 概述

US Stock Analysis 提供一套完整的分析框架,用于评估单只美股。该技能不依赖单一脚本或 API,而是利用 Claude 的网络搜索能力获取实时市场数据,并应用存储在其参考知识库中的结构化分析框架。

<span class="badge badge-free">无需 API</span>

**它解决什么:**
- 把基本面、技术面与估值分析整合进单一工作流
- 按 10 段模板生成结构化投资报告
- 支持股票并排对比(例如 AAPL vs MSFT)
- 提供看多与看空两方面的论证,便于平衡决策
- 通过 WebSearch 获取最新数据 —— 无需手动录入

**四种分析类型:**

| 类型 | 深度 | 最适合 |
|------|------|--------|
| **基础(Basic)** | 快速概览 | 查价、关键指标、近期新闻 |
| **基本面(Fundamental)** | 深入财务剖析 | 业务质量、利润率、负债、估值 |
| **技术面(Technical)** | 图表与指标分析 | 趋势、支撑/阻力、RSI、MACD |
| **综合(Comprehensive)** | 完整投资报告 | 含目标价的买入/持有/卖出建议 |

## 2. 前提条件

- **API 密钥:** 无需
- **数据来源:** Claude 使用 WebSearch 与 WebFetch 从 Yahoo Finance、MarketWatch、Seeking Alpha、SEC 文件等公开来源获取实时股票数据
- **无 Python 依赖** —— 这是纯对话式技能,没有 CLI 脚本

> US Stock Analysis 完全通过自然语言提示词工作。只需描述你想分析什么,Claude 即会处理数据收集、分析与报告生成。

## 4. 工作原理

1. **判定分析类型** —— Claude 识别四种分析类型(基础、基本面、技术面、综合)中哪一种最匹配你的请求。
2. **收集市场数据** —— WebSearch 查询从可信来源收集当前价格、财务、分析师评级、技术指标与近期新闻。
3. **加载参考框架** —— 按需加载相关知识库:`fundamental-analysis.md`(业务质量评估)、`technical-analysis.md`(指标解读)、`financial-metrics.md`(比率定义)、`report-template.md`(输出结构)。
4. **应用分析框架** —— 数据按结构化标准评估:盈利趋势、现金流质量、资产负债表强度、竞争优势、相对同业估值与技术信号。
5. **生成结构化输出** —— 报告遵循 10 段模板:执行摘要、公司概览、投资论点(看多/看空)、基本面分析、估值分析、技术分析、风险评估、催化剂、建议与结论。

## 5. 使用示例

### 示例 1:快速概览

**提示词:**
```
MSFT 当前价格和关键指标是多少?
```

**你会得到:** 简洁摘要,含当前价格、市值、P/E、EPS、营收增速、52 周区间、年初至今表现,以及任何值得注意的近期新闻。

**为何有用:** 开盘前或研究中快速查看,无需完整报告即可获得快照。

---

### 示例 2:基本面分析

**提示词:**
```
分析 NVDA 的财务 —— 营收趋势、利润率、负债与业务质量
```

**你会得到:** 对 3-5 年营收与盈利趋势、盈利能力指标(毛利/营业/净利率)、资产负债表分析(资产负债率、流动比率、现金状况)、竞争优势与成长可持续性评估的深入剖析。

**为何有用:** 评估业务基本面是否支撑当前股价,识别财务报表中的优势与危险信号。

---

### 示例 3:技术分析

**提示词:**
```
AMZN 技术分析 —— 趋势、支撑/阻力与指标
```

**你会得到:** 当前趋势方向与强度、关键支撑与阻力位、均线位置(20/50/200 日)、RSI 读数、MACD 状态、成交量趋势,以及任何正在形成的明显图形形态。

**为何有用:** 基于技术信号识别最佳进出场时机,以价格行为背景补充基本面分析。

---

### 示例 4:综合报告

**提示词:**
```
给我一份 META 的完整投资报告,附买入/持有/卖出建议
```

**你会得到:** 完整的 10 段报告,涵盖业务概览、投资论点(看多与看空)、基本面分析、含同业对比与公允价值估计的估值分析、技术分析、风险评估、即将到来的催化剂,以及含目标价、确信度与入场策略的最终建议。

**为何有用:** 提供可获得的最详尽分析,适合做实际投资决策或向他人陈述。

---

### 示例 5:同业对比

**提示词:**
```
对比 AAPL vs MSFT —— 现在哪个是更好的投资?
```

**你会得到:** 并排对比表,涵盖商业模式、财务指标(营收、利润率、增速)、估值比率(P/E、PEG、EV/EBITDA)、资产负债表强度、技术定位、相对强度分析,以及哪只股票更具吸引力的建议与组合配置提议。

**为何有用:** 用跨各维度的量化对比,直接回答“我该买哪只?”的问题。

---

### 示例 6:财报后影响

**提示词:**
```
GOOGL 刚发布财报 —— 分析结果与市场反应
```

**你会得到:** 对比共识的盈利超预期/不及预期分析、营收与 EPS 趋势、指引变化、管理层评论要点、市场反应(价格波动、成交量),以及对该股前景的前瞻含义。

**为何有用:** 快速评估一份财报是否改变投资论点,对决定加仓、持有还是减仓尤为重要。

---

### 示例 7:估值评估

**提示词:**
```
特斯拉被高估了吗?看 P/E、PEG、EV/EBITDA,并与汽车同业对比
```

**你会得到:** 当前估值指标与历史均值及同业组(传统车企、纯电动、科技)的对比、用多种方法的公允价值估计、安全边际计算,以及价格中隐含的成长预期是否现实的评估。

**为何有用:** 穿透叙事驱动的定价,用多种估值视角判断该股在当前水平是否具备价值。

## 6. 解读输出

综合报告遵循结构化的 10 段模板:

1. **执行摘要** —— 可独立成段的 2-3 段概览,含建议、核心论点与估值小结。
2. **公司概览** —— 业务描述、市场地位与关键统计表。
3. **投资论点** —— 看多理由(3-5 个带证据的正面因素)与看空理由(3-5 个带概率评估的风险)。
4. **基本面分析** —— 业务质量、财务表现表与成长分析。
5. **估值分析** —— 当前指标表、同业对比表、公允价值区间与安全边际。
6. **技术分析** —— 趋势、支撑/阻力位、指标读数与图形形态。
7. **风险评估** —— 公司特定风险(按优先级)、市场/宏观风险与缓释因素。
8. **催化剂与时间线** —— 可能推动股价的即将到来事件(财报、产品发布、监管)。
9. **建议** —— 买入/持有/卖出评级、确信度(高/中/低)、带时间框架的目标价与入场策略。
10. **结论** —— 要点小结与监控触发条件。

报告保存到 `reports/` 目录,命名约定为 `stock_analysis_[TICKER]_[DATE].md`。

## 7. 技巧与最佳实践

- **明确指定分析类型。** 说“AAPL 基本面分析”会得到聚焦的财务深剖,而“分析 AAPL”默认是较宽泛的概览。请明确你需要什么。
- **在多股之间抉择时请求对比。** 同业对比格式是评估备选最高效的方式,生成的并排表能让差异一目了然。
- **结合财报日历择时。** 财报前做综合分析以确立预期,财报后再做影响分析以更新论点。
- **若你有自己的框架,请请求具体指标。** 问“NVDA 的 ROIC、自由现金流收益率与营收增速是多少?”比一份完整报告更快拿到目标数据。
- **用于持仓复盘。** 定期对持仓运行更新分析,检查最初的投资论点是否仍成立。
- **检查数据时效。** 该技能会在输出中标注数据来源与日期。对快速变化的情形,确认数据反映最近一个季度。

## 9. 故障排查

### 数据似乎过时

**原因:** WebSearch 对某些指标可能返回缓存或较旧的结果。

**修复:** 让 Claude 专门搜索最近一个季度:用“搜索 AAPL Q1 2026 财报结果”而非“AAPL 财务”。指定季度与年份能提升数据新鲜度。

### 缺少技术指标

**原因:** 某些技术数据(精确的 RSI 值、MACD 柱状图)仅靠网络搜索可能无法获得。

**修复:** 如需精确技术读数,结合使用 Technical Analyst 技能(用真实图表图像),或指定“搜索 TradingView AAPL RSI”以定向技术数据来源。

### 对比遗漏了某指标

**原因:** 并非所有公司都披露相同指标(例如订阅收入 vs 产品收入)。

**修复:** 让 Claude 标注哪些指标不可直接对比,并给出最接近的等价项。指定“用 P/E、PEG、EV/EBITDA、FCF 收益率与营收增速对比 AAPL vs MSFT”可确保得到你想要的确切指标。

### 报告太长或太短

**原因:** 对含糊请求,该技能默认输出综合报告。

**修复:** 明确指定分析类型。“快速概览”产出简洁摘要;“综合报告”产出完整 10 段模板;“只要估值部分”则只给你所需。

## 10. 参考资料

### 分析类型

| 类型 | 触发短语 | 关键输出 |
|------|----------|----------|
| **基础** | “快速概览”“关键指标”“现价多少” | 价格、市值、P/E、近期新闻 |
| **基本面** | “财务”“业务质量”“是否高估” | 营收趋势、利润率、负债、估值比率 |
| **技术面** | “技术分析”“趋势”“支撑位” | 均线、RSI、MACD、图形形态 |
| **综合** | “完整报告”“完整分析”“我该投资吗” | 含买入/持有/卖出建议的 10 段报告 |

### 参考知识库

| 文件 | 何时加载 | 内容 |
|------|----------|------|
| `references/fundamental-analysis.md` | 基本面或综合分析 | 业务质量评估、财务健康分析、估值框架、风险评估、危险信号 |
| `references/technical-analysis.md` | 技术面或综合分析 | 指标定义、图形形态、支撑/阻力概念、分析工作流 |
| `references/financial-metrics.md` | 任何需要比率定义的分析 | 全部关键指标及公式:盈利、估值、成长、流动性、杠杆、效率、现金流 |
| `references/report-template.md` | 综合报告或对比 | 完整报告结构、格式指引、章节模板、对比格式 |

### 使用的数据来源

| 数据类型 | 首选来源 |
|----------|----------|
| 价格与交易数据 | Yahoo Finance、Google Finance |
| 财务报表 | SEC EDGAR(10-K、10-Q)、公司 IR 页面 |
| 分析师评级 | MarketWatch、Seeking Alpha、Bloomberg |
| 技术数据 | TradingView、StockCharts |
| 新闻与动态 | CNBC、Reuters、WSJ |
