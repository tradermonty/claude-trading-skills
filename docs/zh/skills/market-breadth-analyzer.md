# Market Breadth Analyzer
{: .no_toc }

## 1. 概述

Market Breadth Analyzer 回答每个股票投资者在加仓前都该问的问题:“这波上涨是广泛参与的,还是少数几只超大盘股在掩盖底层的疲弱?”

**它解决什么:**
- 在 6 个加权分量上给出单一综合评分(0-100,100 = 健康)
- 用完全可复现的量化方法,消除主观看图
- 跨多次运行跟踪评分历史,识别改善或恶化趋势
- 把健康区间直接映射到建议的股票敞口水平
- 优雅处理缺失数据,自动重新分配权重

**核心能力:**
- 6 分量自动加权评分:宽度水平与趋势(25%)、8MA 与 200MA 交叉(20%)、峰/谷周期(20%)、看跌信号(15%)、历史分位(10%)、S&P 500 背离(10%)
- 5 个健康区间:强(80-100)、健康(60-79)、中性(40-59)、转弱(20-39)、危急(0-19)
- 多窗口背离检测(20 日 + 60 日),衡量 S&P 500 价格与宽度之间的背离
- 滚动评分历史(最多 20 条)及趋势标签:改善、稳定、恶化

<span class="badge badge-free">无需 API</span>

- **API 密钥:** 无需 —— 使用 GitHub Pages 上免费提供的 CSV 数据
- **Python 3.9+:** 运行分析脚本所需
- **网络连接:** 需要从 TraderMonty 的公开 GitHub Pages 站点获取 CSV 数据
- **无额外 Python 依赖** —— 仅使用标准库

## 3. 快速开始

运行分析脚本:

```bash
python3 skills/market-breadth-analyzer/scripts/market_breadth_analyzer.py --output-dir reports/
```

脚本会获取最新 CSV 数据,对全部 6 个分量评分,并输出带区间分类与建议行动的综合健康评分。这就是上手所需的一切。

---

## 4. 工作原理

1. **获取数据** —— 脚本从 TraderMonty 的 GitHub Pages 下载两个 CSV 文件:明细文件(从 2016 年至今约 2,500 行)与汇总文件(8 个聚合指标)。
2. **校验新鲜度** —— 若最新数据点超过 5 天,报告中会包含数据过期警告。
3. **对 6 个分量评分** —— 每个分量基于最新数据行加历史背景,在 0-100 区间独立评分。数据不足的分量被排除,其权重按比例重新分配给其余分量。
4. **计算综合评分** —— 所有可用分量评分的加权平均得出最终 0-100 综合分。
5. **分类健康区间** —— 综合分映射到 5 个区间之一(强、健康、中性、转弱、危急),每个区间对应一个建议的股票敞口范围。
6. **跟踪历史** —— 评分被追加到滚动历史文件。当存在多次观测时,趋势(改善 / 稳定 / 恶化)由首末条目之间的差值计算。
7. **输出报告** —— JSON 与 Markdown 文件保存到输出目录。

**权重重新分配:** 若任一分量缺数据(例如背离分析的行数太少),其权重被排除,其余权重按比例重新缩放,使其仍合计为 100%。

## 5. 使用示例

### 示例 1:早盘市场健康检查

**提示词:**
```
现在市场宽度健康吗?
```

**发生了什么:** 脚本以默认设置运行,获取最新 CSV,返回综合评分、健康区间与建议的股票敞口。这是开盘前的快速日检。

**为何有用:** 几秒内确立市场环境基线。处于强区间(80-100)意味着成长与动量策略得到良好支撑;转弱区间(20-39)则提示该收紧止损、提高现金。

---

### 示例 2:上涨广度评估

**提示词:**
```
S&P 500 创历史新高了。这波上涨是广泛参与还是狭窄的?
```

**发生了什么:** 分析器评估宽度水平与趋势分量(C1)以及 S&P 500 背离分量(C6)。若指数上涨但宽度下降,背离评分会急剧下降,综合分会反映这一风险。

**为何有用:** 在狭窄龙头带动下创新高,是市场见顶最可靠的预警信号之一。本分析把看图者直观看到的东西量化出来。

---

### 示例 3:狭窄龙头预警检测

**提示词:**
```
当前上涨中参与的股票变少了吗?检查宽度是否在收窄。
```

**发生了什么:** 分析器检查 8MA 相对 200MA 的水平,检测 20 日与 60 日窗口的看跌背离,并标记任何“早期预警”信号 —— 即短期宽度恶化、而结构性宽度看起来仍健康。

**为何有用:** 宽度收窄往往领先回调数周。检测到早期预警标志,可在下跌变得显而易见之前调整组合敞口。

---

### 示例 4:S&P 500 背离分析

**提示词:**
```
S&P 500 的价格走势与市场宽度之间存在背离吗?
```

**发生了什么:** 分量 C6 使用多窗口分析(20 日权重 40%,60 日权重 60%),衡量价格与宽度是否同向。识别四种形态:健康同步、危险的狭窄市场、看涨背离(潜在底部)与持续下跌。

**为何有用:** 双窗口方法能同时捕捉新出现的短期背离与结构性的中期背离。最危险的形态 —— S&P 上涨而宽度下降 —— 曾出现在 2000、2007 与 2021 年的重大顶部之前。

---

### 示例 5:历史分位对比

**提示词:**
```
当前宽度相比过去 10 年处于什么位置?
```

**发生了什么:** 分量 C5 把当前 8MA 读数放入完整的历史分布(2016 年至今)。报告显示分位排名,并标记宽度是接近平均峰值水平(过热,-10 调整)还是接近平均谷值水平(逆向机会,+10 调整)。

**为何有用:** 背景很重要。8MA 为 0.55 可能感觉中性,但若历史峰值平均为 0.73,则仍有显著上行空间。反之,接近历史谷值水平(约 0.23)的读数,历史上一直是极佳的长期入场点。

---

### 示例 6:用区间切换做敞口决策

**提示词:**
```
上周宽度处于转弱区。它改善到足以加仓了吗?
```

**发生了什么:** 分析器计算今日评分并与存储的历史对比。趋势标签(改善、稳定或恶化)加上区间切换(例如 转弱→中性)指导敞口决策。若评分从 39 升到 42,区间从转弱变为中性,提示可把股票敞口从 40-60% 谨慎提升到 60-75%。

**为何有用:** 区间切换是可执行的信号。基于历史的趋势确保你响应的是市场宽度的真实转变,而非单次读数带来的噪声。

## 6. 解读输出

执行后,脚本生成两个文件,并在控制台打印摘要:

1. **综合评分** —— 代表整体市场健康度的单一 0-100 数值(100 = 最健康)。
2. **健康区间** —— 5 个区间之一,附建议的股票敞口:强(90-100%)、健康(75-90%)、中性(60-75%)、转弱(40-60%)、危急(25-40%)。
3. **分量拆解** —— 6 个分量各自的评分、有效权重与简要解读。
4. **数据质量标签** —— 完整(6/6 分量)、部分(4-5/6)或有限(0-3/6),表明置信水平。
5. **评分历史与趋势** —— 历次评分及趋势方向(改善 / 稳定 / 恶化)。
6. **建议行动** —— 基于当前区间的具体步骤(例如强区间下“满仓配置;偏好成长/动量”)。
7. **需关注的关键水平** —— 重要阈值,如 8MA 上穿 200MA 或接近历史峰/谷均值。

### 健康区间速查

| 评分 | 区间 | 股票敞口 | 行动 |
|------|------|----------|------|
| 80-100 | 强 | 90-100% | 满仓,偏好成长/动量 |
| 60-79 | 健康 | 75-90% | 正常运营 |
| 40-59 | 中性 | 60-75% | 精选配置,收紧止损 |
| 20-39 | 转弱 | 40-60% | 获利了结,提高现金 |
| 0-19 | 危急 | 25-40% | 保本,关注是否筑底 |

## 7. 技巧与最佳实践

- **每天开盘前运行。** 宽度状况是渐变的,但每日检查能确保你尽早捕捉区间切换。单次运行仅需几秒。
- **与 Breadth Chart Analyst 技能配合。** 分析器给出量化评分;图表分析师提供定性的视觉形态识别。两者结合提供完整的宽度图景。
- **留意早期预警标志。** 当 20 日背离窗口转为看跌、而 60 日窗口仍健康时,短期恶化已开始。这是最早的可执行信号。
- **不要对单次读数反应过度。** 用评分历史趋势(改善 / 稳定 / 恶化)在改变敞口前确认方向。单日评分回落不值得大改组合。
- **关注权重重新分配的说明。** 若报告显示可用分量少于 6 个,请相应谨慎解读综合分。“部分”数据质量标签意味着评分可靠性较低。
- **把基于区间的敞口当作指引,而非铁律。** 建议的股票敞口范围只是起点。请根据你个人的风险承受度、组合构成与确信度调整。

## 8. 与其他技能组合

| 工作流 | 如何组合 |
|--------|----------|
| **每日市场评估** | 先运行 Market Breadth Analyzer 得到量化评分,若区间接近切换,再用 Breadth Chart Analyst 做视觉确认 |
| **板块轮动择时** | 当宽度从强转向转弱时,用 Sector Analyst 识别防御性轮动候选 |
| **仓位规模调整** | 把健康区间纳入 Position Sizer 决策:在转弱/危急区用更紧的风险(0.5%),在健康/强区用标准风险(1%) |
| **成长股筛选** | 在强/健康区间,CANSLIM 与 VCP 筛选的成功率更高。在转弱区间,转而偏好股息与价值筛选 |
| **顶部检测** | 若宽度处于转弱且 Market Bubble Detector 显示橙/红,把这视为见顶状况的强确认 |

## 9. 故障排查

### “数据似乎过期”警告

**原因:** CSV 中的最新行超过 5 天,通常是因为 TraderMonty 的 GitHub Actions 流水线未运行。

**修复:** 这是警告而非错误。分析仍会使用最近可用数据运行。到[源仓库](https://github.com/sydcurie/market-breadth-analysis)查看更新状态。市场假期期间出现过期是正常的。

### “Component X excluded (insufficient data)” 消息

**原因:** 某分量因数据缺少必要的行或标记(例如数据集中未检测到峰/谷标记)而无法评分。

**修复:** 这会通过权重重新分配自动处理。其余分量按比例吸收被排除的权重。检查输出中的数据质量标签:“完整”表示 6 个分量都评了分;“部分”或“有限”表示置信度降低。

### 获取 CSV 时网络报错

**原因:** 无法访问 GitHub Pages(网络问题、企业防火墙,或托管服务临时宕机)。

**修复:** 检查网络连接。尝试在浏览器中直接打开数据 URL:`https://sydcurie.github.io/market-breadth-analysis/market_breadth_data.csv`。若被防火墙拦截,可手动下载 CSV 并传入其本地路径(脚本当前预期 URL;你可能需要修改获取逻辑以支持本地文件)。

### 市场上涨,评分却意外偏低

**原因:** S&P 500 可能是在狭窄龙头带动下上涨(少数大盘股推动指数)。即使指数看起来强,宽度评分也正确地反映了参与的疲弱。

**修复:** 这是设计预期 —— 也正是宽度分析的核心价值。检查分量 C6(S&P 500 背离)以确认狭窄市场状况。

## 10. 参考资料

### CLI 参数

| 参数 | 必需 | 默认 | 说明 |
|------|------|------|------|
| `--detail-url` | 否 | TraderMonty GitHub Pages URL | 明细 CSV 的 URL(market_breadth_data.csv) |

| `--output-dir` | 否 | `.`(当前目录) | JSON 与 Markdown 报告的输出目录 |

### 6 分量评分概览

| # | 分量 | 权重 | 衡量内容 |
|---|------|------|----------|
| 1 | 宽度水平与趋势 | 25% | 当前 8MA 水平 + 200MA 趋势方向 + 8MA 方向修正 |
| 2 | 8MA 与 200MA 交叉 | 20% | 通过 MA 间距与方向衡量动量 |
| 3 | 峰/谷周期 | 20% | 在宽度周期中的位置(早期复苏、成熟、峰后等) |
| 4 | 看跌信号 | 15% | 经回测的看跌信号标志,按背景调整 |
| 5 | 历史分位 | 10% | 当前 8MA 相对完整历史分布 |
| 6 | S&P 500 背离 | 10% | 多窗口(20 日 + 60 日)价格与宽度背离 |

### 关键宽度阈值

| 8MA 水平 | 解读 |
|----------|------|
| > 0.70 | 非常强 —— 广泛上涨 |
| > 0.60 | 健康 —— 高于平均参与度 |
| > 0.50 | 中性 —— 约一半股票参与 |
| > 0.40 | 转弱 —— 低于平均参与度 |
| < 0.40 | 极度疲弱 —— 可能正在筑谷 |
| < 0.20 | 危机水平 —— 罕见,常先于重大底部 |

### 输出文件

| 文件 | 说明 |
|------|------|
| `market_breadth_YYYY-MM-DD_HHMMSS.json` | 含全部分量评分、综合分与元数据的结构化 JSON |
| `market_breadth_YYYY-MM-DD_HHMMSS.md` | 人类可读的 Markdown 报告 |
| `market_breadth_history.json` | 综合评分的滚动历史(最多 20 条) |
